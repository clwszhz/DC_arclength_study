# Homotopy 方法阅读笔记

来源：https://www.sfu.ca/~ljilja/cnl/projects/Homotopy/

## 1. 同伦方法要解决什么问题

非线性电路的 DC 工作点可以写成方程组 `F(x) = 0`。普通 Newton 方法收敛很快，但需要初值足够接近真实解；对于多解电路或强非线性晶体管电路，初值不好选时容易失败。

同伦方法把原问题嵌入到一个带参数 `lambda` 的方程 `H(x, lambda) = 0` 中。`lambda = 0` 时问题容易求解，`lambda = 1` 时回到原方程。

```text
原问题：F(x) = 0
同伦问题：H(x, lambda) = 0
起点：H(x, 0) = 0
目标：H(x, 1) = F(x) = 0
```

## 2. 固定点同伦的基本形式

文章使用的固定点同伦形式为 `H(x, lambda) = (1 - lambda)G(x - a) + lambda F(x)`。其中 `a` 是随机起始向量，`G` 是对角缩放矩阵，`F(x)` 是原始电路方程。

当 `lambda = 0` 时，方程变成 `G(x - a) = 0`，容易得到起点 `x = a`；当 `lambda = 1` 时，方程变成 `F(x) = 0`，即原始电路 DC 工作点方程。

```text
lambda = 0: H = G(x - a)
lambda = 1: H = F(x)
```

## 3. 弧长续接的作用

直接把 `lambda` 从 0 增加到 1 可能失败，因为同伦路径可能折回。在折返点附近，路径继续前进时 `lambda` 反而减小，单纯用 `lambda` 作为自变量会丢失路径。

弧长续接把 `x` 和 `lambda` 都看成路径弧长 `s` 的函数，即沿着曲线本身前进，而不是只沿着 `lambda` 方向前进。

```text
普通续接：x = x(lambda)
弧长续接：x = x(s), lambda = lambda(s)
```

## 4. 电路 DC 工作点中的整体流程

文章中的流程是先由 Parser 根据 SPICE 网表生成节点方程或改进节点方程，以及对应的雅可比矩阵；然后 MATLAB 中的 `pchomotopy` 用 predictor-corrector 方法追踪同伦路径。

对于 Chua 四晶体管电路，文章使用改进节点方程建立 18 个未知量，其中 `x1` 到 `x14` 是节点电压，`x15` 到 `x18` 是独立电压源电流。同伦路径与 `lambda = 1` 的交点给出 DC 工作点。

```text
SPICE 网表
  -> 生成 MNA 方程 F(x)
  -> 构造 H(x, lambda)
  -> predictor-corrector 追踪路径
  -> 在 lambda = 1 处得到解
```

## 5. 为什么一条同伦路径可能给出多个稳态点

不动点同伦 `H(x, lambda) = 0` 中，未知量是 `x` 和 `lambda`，方程数量比未知量少一个，因此它的解集通常是一条曲线。`lambda = 1` 时，`H(x, 1) = F(x)`，所以同伦曲线与 `lambda = 1` 的每个交点都是原方程 `F(x) = 0` 的一个解。

如果算法只把 `lambda` 从 0 单调增加到 1，通常只能得到一个交点；弧长续接则沿曲线本身前进，允许 `lambda` 增大、减小或折回。因此同一条同伦曲线可能多次穿过 `lambda = 1`，每次穿过都可能对应一个不同的 DC 工作点。

```text
同伦路径：H(x, lambda) = 0
目标平面：lambda = 1

第一次相交 -> 解 1
第二次相交 -> 解 2
第三次相交 -> 解 3
```

## 6. `deval` 中 QR 分解的作用

`deval` 要求同伦路径的切向量 `z`，它满足 `DH * z = 0`。代码先做 QR 分解 `DH = Q * R`，由于 `Q` 是正交可逆矩阵，`Q * R * z = 0` 等价于 `R * z = 0`，所以只需要用 `R` 来求零空间方向。

```text
DH * z = 0
DH = Q * R
Q * R * z = 0
R * z = 0
```

随后固定最后一个分量 `z(n+1) = 1`，把 `R * z = 0` 改写成

```text
R(:,1:n) * z(1:n) = -R(:,n+1)
```

对应代码为：

```matlab
z(1:n) = -R(:,1:n)\R(:,(n+1));
z(n+1) = 1;
```

最后再把 `z` 归一化，并用上一时刻的切向量 `ypold` 判断正负方向。

## 7. `ypold` 用来保持切向量方向连续

`deval(J, y, ypold)` 中的 `ypold` 是上一时刻的切向量。由于 `z` 和 `-z` 都满足 `DH * z = 0`，代码用

```matlab
yp = sign(zn' * ypold) * zn;
```

选择和上一切向量夹角小于 90 度的方向。启动时还没有上一切向量，所以令 `fc(1,1) = 1`，相当于先假设路径从 `lambda` 增大的方向开始走。

## 8. 启动阶段的 predictor-corrector

启动阶段先用当前点切向量预测一步：

```matlab
fp = deval(J, y, fc);
yp = y + h * fp;
```

预测点 `yp` 不一定精确落在同伦曲线上，因此再在 `yp` 附近重新计算切向量 `fc`，并从原点 `y` 重新走一步：

```matlab
fc = deval(J, yp, fp);
yc = y + h * fc;
```

这里的作用不是严格投影回 `H = 0`，而是用预测点附近的新切线修正这一步，同时为后面的高阶方法积累历史切向量。

## 9. `Y` 到 `Zc` 的准备过程

进入主循环前，启动阶段已经积累了若干个历史切向量 `f`。代码先把当前点和这些导数信息整理成 `Y`：

```matlab
d = f * deltas';
Y = [y; h * d];
for i = 1:k-1
    Y = [Y; h * f(:,i)];
end
```

`Y` 可以理解成“当前点 + 若干历史导数”。随后

```matlab
Zc = Q * Y;
```

把这种历史数据格式转换成主循环使用的局部多项式格式。这里的 `Q` 是程序自己构造的变换矩阵，不是 QR 分解里的 `Q`。

## 10. `Zc`、`PI` 和 `Zp` 的含义

`Zc` 可以理解成当前路径附近的局部多项式信息。

```text
Zc =
[
  当前点信息
  一阶导数信息
  更高阶信息
  ...
]
```

如果 `m = 9`、`k = 4`，那么每一块是一个 `9 x 1` 的 `[lambda; x]` 向量，一共有 `k+1 = 5` 块，所以 `Zc` 是 `45 x 1`。

## 11. 多项式平移的 `P(t)` 含义

可以把局部路径近似写成一个多项式：

```text
P(t) = C0 + C1*t + C2*t^2 + ...
```

当前点对应 `t = 0`，下一步对应 `t = 1`。如果走到下一步后重新把当前位置当成原点，令 `t = 1 + u`，则

```text
P(1 + u)
```

就是平移后的多项式。比如二次情形：

```text
P(t) = C0 + C1*t + C2*t^2
P(1+u) = (C0+C1+C2) + (C1+2*C2)u + C2*u^2
```

所以新的系数由旧系数线性组合得到，这个线性变换就是多项式平移矩阵。二项式展开中的 `1, 2, 1`、`1, 3, 3, 1` 等系数来自 Pascal 三角形。

## 12. `PI = kron(..., eye(m))` 的作用

`PI` 是由 Pascal 系数构造的多项式平移矩阵：

```matlab
PI = kron(abs(pascal(k+1,1))', eye(m));
```

`pascal(k+1,1)` 给出平移多项式需要的系数，`eye(m)` 表示对 `y = [lambda; x]` 的每个分量使用同一套平移规则。`kron` 则把标量多项式的平移规则扩展到 `m` 个变量上。

执行

```matlab
Zp = PI * Zc;
```

就是把当前局部多项式向前平移一步，得到预测状态。`Zp(1:m)` 是预测的下一点，`Zp(m+1:2*m)` 是预测的下一点导数信息。

## 13. Corrector 修正预测状态

预测后，代码在预测点重新计算真实切向量：

```matlab
fnpk = deval(J, Zp(1:m), fnpk);
```

然后比较“重新算出的导数信息”和“预测器给出的导数信息”：

```matlab
F = h * fnpk - Zp(m+1:2*m);
Zctemp = Zp + Gt * F;
```

`F` 是导数误差，`Zctemp` 是校正后的状态。也就是说，predictor 给出候选下一点，corrector 用预测点处的新切向量修正这个候选状态。

## 14. 校正后还要做误差验收和步长控制

校正完成后，算法并不会立刻接受这一步，而是先比较预测点和校正点的差：

```matlab
error = W * (Zctemp(1:m) - Zp(1:m));
delta = norm(error, 'inf');
tau = tol(1) * max(norm(y, 'inf'), 1.0);
```

若 `delta < tau`，说明这一步误差可接受，才更新 `y`、`lambda`、`s` 和轨迹数据。若误差太大，则减小步长重新预测。

如果这一步跨过 `lambda = 1`，但没有刚好落在 `lambda = 1` 附近，代码进入 `endgame`，用 Hermite 插值估计更合适的步长，使下一步尽量落到 `lambda = 1`，因为那里才对应原方程 `F(x) = 0` 的解。

## 15. 步长改变后要缩放 `Zc`

每一步结束后代码会根据误差调整下一步步长 `h`。如果新旧步长比例为

```matlab
alpha = h / hold;
```

那么 `Zc` 中和步长相关的高阶块也要同步缩放：

```matlab
for i = 1:k
    Zc((i*m+1):(i+1)*m) = alpha^i * Zc((i*m+1):(i+1)*m);
end
```

直观上，一阶导数块跟 `h` 成正比，二阶块跟 `h^2` 成正比，因此第 `i` 阶块要乘 `alpha^i`。
