# demo 改动记录

## 1. 合并程序入口

原来工程中线性直流求解和同伦求解是两个入口。现在统一到 `src/main.cpp`：程序先读取网表，再判断电路中是否含有 BJT。没有 BJT 时走线性流程；含有 BJT 时走非线性同伦流程。

这样改的原因是：线性电路和非线性电路本质上都是求静态工作点，只是线性电路可以直接解 `A*x=b`，非线性电路需要解 `F(x)=0`。

## 2. 扩展网表功能

在原来 `R/I/V` 的基础上，加入了 BJT 相关描述：

```text
Q      BJT 器件
.MODEL BJT 模型参数
.START 同伦初始点
```

这样 Schmitt 电路不需要手写专门方程，而是可以通过网表描述电路结构，再由程序自动生成 MNA 方程。`.START` 用来对齐 MATLAB 代码中的初始向量，方便比较每一步结果。

## 3. 新增非线性 MNA

原来的 `Mna.cpp` 生成固定的线性矩阵 `A,b`。新增的 `Nonlinear.cpp` 改为根据当前未知量 `x` 计算：

```text
F(x)  方程残差
J(x)  雅可比矩阵
```

电阻、电流源、电压源、BJT 都通过 stamp 加入 `F` 和 `J`。这样程序从“线性 MNA”扩展为“通用非线性 MNA”，后续支持更多非线性器件时也可以继续沿用这个框架。

## 4. 修正 BJT 电流方向

对照 MATLAB 代码后，发现 C++ 里最开始的 BJT 电流方向和 MATLAB 参考方程不一致。现在已经把 BJT 的端口电流和对应偏导改成与 MATLAB 的 Ebers-Moll 写法一致。

这个改动很关键：如果电流方向不一致，即使同伦算法本身没有问题，求解的也不是 MATLAB 里的同一组方程。

## 5. 改进同伦求解流程

`Homotopy.cpp` 已经改成全程伪弧长 continuation：从 `lambda=0, x=a` 出发，先计算同伦曲线切线，再用 predictor 沿 `[lambda,x]` 空间走一小段，最后用 Newton corrector 解增广方程，把预测点拉回 `H(lambda,x)=0` 曲线上。

如果 corrector 遇到奇异矩阵或不收敛，就缩小弧长步长重试，而不是切换回自然参数方法。只有路径到达或越过 `lambda=1` 后，程序才会固定 `lambda=1`，用原始 `F(x)=0` 做最终 Newton 验证。

## 6. 对齐 MATLAB 的同伦行顺序

这是 Schmitt1 能跑通的关键改动。

标准 MNA 的行顺序通常是：

```text
节点 KCL 方程在前
电压源约束方程在后
```

但 MATLAB 示例中，为了使用固定点同伦

```text
H(lambda,x) = (1-lambda)*G*(x-a) + lambda*F(x)
```

它把电压源约束方程放在对应节点电压的位置，把对应节点 KCL 放到电压源电流变量的位置。线性求解时这只是行交换，结果不变；但同伦中 `G*(x-a)` 是按行相加的，行顺序不同会改变整条同伦路径。

因此 C++ 现在只在构造同伦函数时做行重排：标准 MNA 残差仍保持原来的通用形式，`H(lambda,x)` 内部的 `F/J` 行顺序则对齐 MATLAB。这样既保留通用 MNA，又避免 Schmitt1 跑到错误分支。

## 7. 新增 Schmitt1 网表示例

新增 `examples/schmitt1.cir`，按照 MATLAB 中 Schmitt1 的电路结构写成通用网表。这个例子用于验证：只要器件模型支持 BJT，Schmitt 电路就可以自然列出方程，而不是给某个电路单独硬编码公式。

## 9. 后续重点

接下来可以继续做两件事：

```text
1. 导出 lambda-x 路径 CSV，画出类似 MATLAB Result.jpg 的路径图
2. 如果要找多个解，再复现 MATLAB 的“lambda 多次穿过 1 时记录多个解”的逻辑
```

目前代码的核心进展是：已经从“针对某个电路手写方程”推进到“从网表自动生成非线性 MNA 方程”，并且 Schmitt1/Schmitt2 都能通过同伦追踪到有效工作点。
